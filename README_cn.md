# 盒马时钟

本项目分叉自 [tpunix](https://github.com/tpunix) 的原版 [HMCLOCK](https://github.com/tpunix/HMCLOCK) 项目。

---

![设备展示](./images/device.png)

## 项目简介

本项目利用超市淘汰下来的 **盒马价签** 硬件，实现一个功能简单但实用的电子时钟，具有如下功能：

* ✅ 显示时间与日期
* ✅ 显示农历、节气与节假日
* ✅ 显示电池电量
* ✅ 蓝牙对时
* ✅ 时间校准
* ✅ 蓝牙 OTA 更新

---

## 更新日志

### 0xA50f0008

* 移除了蓝牙长值写入分发逻辑中一段无法执行到的重复分支；
* 为蓝牙设置时钟命令（`0x91`）新增了取值范围校验，避免非法写入触发农历日期查表时的数组越界读取；
* 为设置时钟、时间校准、OTA 等命令新增了最小长度校验，避免过短的写入导致按固定偏移读取越界；
* 删除了未被任何编译目标引用、且已与实际使用的驱动产生分歧的重复 Flash/OTA 驱动文件 `spi_flash_hwctl.c`（该文件在 OTA 完成时缺少复位调用）；
* OTA 升级流程新增了对接收固件的 CRC32 校验，只有校验通过才会复位进入新固件；校验失败时会作废新镜像的头部，防止意外复位（看门狗、掉电等）后引导到写入不完整的固件。

## 编译与烧写

请先下载 DA14585 的 [SDK](https://www.renesas.com/en/document/swo/sdk60221401-da1453x-da145856 "SDK") 包，目前使用 **6.0.22.1401** 版本。

1. 将本项目放置在：

   ```
   SDK_PATH/projects/target_apps/ble_examples
   ```

2. 使用Keil打开项目进行编译。

3. 编译完成后：

   * **调试模式运行一次**，固件会自动写入 Flash；
   * 或者使用 **SmartSnippets Toolbox** 将固件下载到 RAM 运行一次也可；
   * 如果使用的是 CMSIS-DAP/ST-Link 调试器而非 J-Link，也可以用 **OpenOCD** 烧写（见下文）。

### 使用 OpenOCD 烧写

DA14585/586 没有内部 Flash —— 固件通常运行在 RAM 中，由调试器直接加载，或由 ROM 启动引导程序从外部 SPI Flash/OTP 拷贝而来。`Keil_5\openocd_da14585.cfg` 通过 SWD 使用 CMSIS-DAP 兼容调试器实现这种“加载到 RAM 并运行”的方式（已在本项目 Keil 的 `CMSIS_AGDI` 调试驱动所配置的同一个调试器上测试通过）。

安装 OpenOCD（例如 `winget install xpack-dev-tools.openocd-xpack`），然后在 `Keil_5` 目录下执行：

```
openocd -f openocd_da14585.cfg -c "ram_load_run out_DA14585/Objects/ble_app_peripheral_585.axf" -c shutdown
```

该命令会暂停内核、将 `.axf` 写入 `0x07fc0000` 处的 RAM、根据中断向量表设置 `SP`/`PC`，然后恢复运行——效果等同于 Keil 的 RAM 调试流程。和该流程一样，固件自身的首次启动逻辑会把自己写入外部 SPI Flash，因此无需额外的烧写步骤即可在断电后保持。

注意事项：

* OpenOCD 没有针对该芯片外部 SPI Flash/OTP 的 flash 驱动，因此 `program`/`flash write_image` 在这里不可用——只有“加载到 RAM”这条路径可行。
* 请使用普通的 `reset halt`，不要加硬件复位加延时：仅内核复位（`SYSRESETREQ`）**不会**重新触发 ROM 的 Flash 到 RAM 拷贝过程（该过程只在上电复位时发生），所以并不存在需要等待的竞争条件。在暂停前加入延时反而可能在内核执行中途捕获，从而触发 HardFault。

## 蓝牙对时

项目内置页面提供 **Web Bluetooth** 功能，可用于手动对时：

* 固件每隔 **整十分钟** 广播一次，持续 **30秒**；
* 广播状态下，屏幕显示蓝牙图标与设备名后缀；
* 打开页面点击 “连接”，选中设备后，按 “同步时间” 即可完成同步。

## 时间校准

根据观察，默认状态下，时钟大概每天会快两秒左右。通过微调定时器的间隔，可以让时钟更精确。
在时间同步后，间隔两到三周，做两次校准，基本上就可以让时钟精准运行了。

* 设备链接后，按"时间校准"按钮，然后等待一分钟左右即可。
* 校准后请立即做一次时间同步。

首次上电会打开配对页面，扫码跳转网页可以进行配对。

### Flash存储信息

#### 屏幕引脚配置（地址 `0x39000`）

示例：

```
09 01 FF FF FF FF FF FF 21 22 10  01  20  07  11  23
                        CS ?? RST CLK SDI DC BUSY PWR
```

* 第一个字节 `09`：屏幕类型
* 第二个字节 `01`：引脚配置启用标志（非 `01` 表示无效）

#### 屏幕分辨率等信息（地址 `0x3a000`）

示例：

```
00 25 00 00 92 fa a8 fe 00 01 80 00 28 01 04 00
                          0080  0128          128x296  BWR
```

## 其他说明

* 原版价签固件存放在 Flash 的 SUOTA 区域；
* 大多数价签使用 OTP 启动器，但会从 Flash 继续加载固件；
* Flash 中包含屏幕 and 引脚配置，因此新固件无需硬编码这些信息；
* 原生固件无法被蓝牙扫描到，因此难以通过 OTA 更新；
* 价签多数电池电量不足，因此建议拆机替换供电。
